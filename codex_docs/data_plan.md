# 데이터 스플릿 / 샘플링 / B버전 접근 결정

## 스플릿
- 방식: 유저별로 8/1/1 랜덤 분할 (seed 고정). 각 유저의 기록을 섞어 train/val/test로 나눔.
- 아이템 cold case: val/test에만 등장하는 아이템 허용. 보고서에 명시.

## 네거티브 샘플링
- A버전: popularity-aware 혼합. 없는 user-item 중 인기 아이템 위주로 샘플링(예: 70% popular, 30% uniform) 후 배치별 neg 생성.
- B버전: 평점<4인 관측값을 음성으로 사용. 없는 조합은 음성에 포함하지 않음(방법1).

## B버전 모델 형태
- 이진 분류(≥4 추천)와 회귀(평점 예측 후 threshold=4) 두 가지를 모두 실험해본 뒤 성능을 비교하고 선택.
